1. Overview

Aloia PDF Extractor is an AI-powered web application that extracts structured data from PDF documents using Anthropic's Claude AI. It enables business users to define custom data fields, upload one or more PDFs, and receive the extracted information as a downloadable CSV file.

Key capabilities include:

  • AI-Powered Extraction – Uses Claude Sonnet to intelligently read and interpret PDF content.
  • Flexible Schema Definition – Define what to extract using a visual form builder, JSON schema, or plain English.
  • Batch Processing – Upload and process multiple PDFs in a single operation.
  • Cross-Reference / VLOOKUP – Match extracted data against a lookup dataset using fuzzy matching and export the enriched results.
  • CSV Export – All results are downloadable as standard CSV files compatible with Excel, Google Sheets, and other tools.

3. Getting Started

The application guides you through a step-by-step workflow. A progress indicator at the top of the screen shows your current position. The basic extraction flow consists of three steps:

  1. Define Fields – Specify what data you want to extract.
  2. Upload PDFs – Provide the documents to process.
  3. Review Results – Preview and download the extracted data as CSV.

An optional three-step extension enables cross-referencing extracted data against a second dataset (Steps 4–6).

4. Step 1 – Define Extraction Fields

Before uploading documents, you must tell the application what data to extract. Three input modes are available, accessible via tabs at the top of the form:

4.1 Field Builder (Recommended for Most Users)

The visual form builder lets you add fields one at a time. For each field, provide:

  • Field Name – The column header for this data point (e.g., invoice_number).
  • Description / Hint – Guidance for the AI on where or how to find this value (e.g., "Usually at the top right of the first page").

Click Add Field to append a new field. Click the Remove button next to any field to delete it.

4.2 JSON Schema

For advanced users or automated workflows, paste a JSON array defining the fields:

[
  { "name": "vendor_name", "description": "Company name of the vendor" },
  { "name": "invoice_date", "description": "Date the invoice was issued" },
  { "name": "total_amount", "description": "Total amount due" }
]

4.3 Natural Language

Describe what you need in plain English. The AI will interpret your instructions and extract accordingly. This mode is ideal for complex or variable documents. 

Extract the invoice number, vendor name, invoice date,
each line item with its description, quantity, unit price,
and the total amount due.
Tip: Be as specific as possible with field descriptions. The more context you provide, the more accurate the extraction results will be.

5. Step 2 – Upload PDF Documents

After defining your schema, proceed to the upload step:

  • Drag and drop one or more PDF files onto the upload zone, or click to browse your file system.
  • The selected files are listed below the upload zone. Use the Remove button to deselect any file before processing.
  • Click Extract to begin processing.

A progress bar tracks the extraction. Processing time depends on the number and complexity of the uploaded documents.

Note: Each PDF is sent to the AI individually. The results from all documents are merged into a single output CSV. A _source_file column identifies which PDF each row originated from.

6. Step 3 – Review & Download Results

Once extraction is complete, a preview table displays the first 20 rows of extracted data. From here you can:

ActionDescription
Download CSVSave the full dataset as a CSV file to your local machine.
Cross-ReferenceProceed to the cross-reference workflow (Steps 4–6) to match this data against a lookup dataset.
Start OverReset the application and begin a new extraction.

7. Cross-Reference (VLOOKUP) Workflow

The cross-reference feature lets you match your extracted data against a second dataset — similar to a VLOOKUP in spreadsheet applications. This is useful for enriching extracted records with data from a reference source.

4 Define & Upload Lookup Data

Define the schema for your lookup documents using the same three modes (Field Builder, JSON, or Natural Language), then upload the lookup PDFs. The application extracts data from these documents to create the lookup dataset.

5 Configure the Match

Configure how the two datasets should be joined:

  • Source Column – Select the column from your primary extraction to match on.
  • Lookup Column – Select the corresponding column from the lookup dataset.
  • Return Columns – Choose which columns from the lookup dataset to include in the output. Use Select All or Clear for convenience.

Click Run Match to execute.

6 Match Results

The results table shows all matched rows. The match count is displayed above the table. Available actions:

  • Download Matches CSV – Export the enriched dataset.
  • Adjust Match – Return to Step 5 to modify matching criteria.
  • Start Over – Reset the entire workflow.
How Matching Works: The application performs case-insensitive substring matching. A source value matches a lookup value if the source value is contained within the lookup value. A single source row may match multiple lookup rows (one-to-many relationship).

8. API Reference

For integration and automation purposes, the application exposes two API endpoints.

8.1 POST /api/extract

Extracts structured data from uploaded PDFs.

ParameterTypeDescription
pdfsFile[]One or more PDF files (multipart/form-data)
schemastringField definitions or natural language instructions
modestringSchema type: form, json, or nl

Response: text/csv – CSV content with extracted data.

8.2 POST /api/cross-reference

Performs fuzzy matching between two datasets.

ParameterTypeDescription
sourceRowsObject[]Array of source data objects
lookupRowsObject[]Array of lookup data objects
sourceColstringColumn name to match from source
lookupColstringColumn name to match in lookup
returnColsstring[]Columns to include from lookup

Request format: application/json

Response: text/csv – CSV with matched and enriched records.

10. Troubleshooting

IssuePossible CauseResolution
Extraction returns an error Invalid or missing API key Verify ANTHROPIC_API_KEY is set correctly in .env.local and restart the server.
Empty or incomplete results Vague field descriptions Provide more specific descriptions or hints for each field. Use natural language mode for complex documents.
Cross-reference returns no matches No substring overlap between source and lookup values Verify that the selected columns contain comparable data. The match is case-insensitive and uses substring containment.
Application fails to start Missing dependencies Run npm install and ensure Node.js 18+ is installed.
Large PDF processing is slow Document complexity / API latency Consider splitting very large PDFs into smaller sections before uploading.
CSV has unexpected formatting Special characters in extracted data The application handles CSV escaping automatically. If issues persist, check the source PDF content.
